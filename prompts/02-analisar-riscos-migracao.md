# Prompt 02 - Analisar riscos da migração

Use este prompt para identificar riscos técnicos, funcionais e operacionais em uma migração de release do ERP Protheus.

## Prompt

Você é um especialista em ERP Protheus, arquitetura de sistemas, gestão de riscos técnicos e continuidade operacional.

Estou planejando uma migração de release do ERP Protheus.

Contexto:

- Release atual:
- Release destino:
- Banco de dados:
- Sistema operacional:
- Quantidade aproximada de usuários:
- Módulos utilizados:
- Integrações críticas:
- Customizações relevantes:
- Rotinas automáticas:
- APIs expostas:
- Processos críticos:
- Janela de migração:
- Plano de rollback existente:

Com base nesse contexto, gere uma análise de riscos contendo:

1. Tipo de risco
2. Descrição do risco
3. Processo impactado
4. Probabilidade
5. Impacto
6. Criticidade final
7. Ação preventiva
8. Ação de contingência
9. Evidência ou validação recomendada
10. Responsável sugerido

Classifique:

- Probabilidade: Baixa, Média, Alta
- Impacto: Baixo, Médio, Alto
- Criticidade final: Baixa, Média, Alta, Crítica

Separe os riscos em:

- Riscos técnicos
- Riscos funcionais
- Riscos operacionais
- Riscos de integração
- Riscos fiscais
- Riscos de performance
- Riscos de comunicação e governança

Gere a saída em formato de tabela Markdown e finalize com um resumo executivo.
